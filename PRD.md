# Markdown Preview - PRD

## 개요

마크다운 파일을 쉽고 빠르게 프리뷰할 수 있는 웹 애플리케이션

## 목표

- 로컬 마크다운 파일을 브라우저에서 즉시 프리뷰
- 실시간 편집 및 미리보기 지원
- GitHub 스타일 마크다운 + 확장 문법 지원
- **설치 없이 웹에서 바로 사용** (PWA로 오프라인 지원)
- **다양한 스타일 프리셋**으로 개인화
- **초보자 온보딩**: 첫 방문 후 3분 이내 첫 문서 작성 완료

## 성공 지표 (Success Metrics)

### 핵심 KPI
- **사용자 온보딩 성공률**: 첫 방문 후 문서 작성/편집 완료율 80% 이상
- **재방문율**: 7일 내 재방문율 40% 이상
- **평균 세션 시간**: 5분 이상
- **오류 발생률**: 세션당 오류 발생률 1% 이하

### 성능 지표
- **초기 로딩 시간**: 3초 이내 (Lighthouse 기준)
- **렌더링 성능**: 1MB 파일 100ms 이내 렌더링
- **프리뷰 응답 속도**: 입력 후 300ms 이내 프리뷰 갱신

### 만족도 지표
- **사용 편의성**: NPS(Net Promoter Score) 50 이상
- **기능 만족도**: 주요 기능별 만족도 4.0/5.0 이상

## 타겟 사용자

| 사용자 | 주요 용도 |
|--------|-----------|
| 개발자 | README, 기술문서, API 문서 작성 |
| PM/기획자 | 기획서, 스펙 문서, 회의록 작성 |
| 블로거/작가 | 콘텐츠, 포스팅 작성 |
| 학생/입문자 | 마크다운 문법 학습, 노트 작성 |

## 핵심 사용 시나리오

1. **빠른 프리뷰**: .md 파일을 드래그해서 즉시 렌더링 확인
2. **문서 작성**: 처음부터 마크다운 문서 작성
3. **문서 편집**: 기존 .md 파일 수정 및 개선
4. **포맷 학습**: 초보자가 마크다운 문법 배우기

## 사용자 여정 (User Journey)

### 신규 사용자 온보딩 (첫 방문)
```
1. 랜딩 (0초)
   └─> 환영 플레이스홀더 문서 표시 (앱 소개 + 문법 가이드)

2. 빠른 체험 (10초 이내)
   └─> 플레이스홀더 문서에서 마크다운 문법 확인
   └─> 실시간 프리뷰 동작 확인

3. 행동 선택 (30초 이내)
   ├─> [A] "새 문서 시작" 버튼 → 빈 에디터
   ├─> [B] "파일 열기" 버튼 → 기존 .md 파일 업로드
   └─> [C] 플레이스홀더 수정 → 바로 편집 시작

4. 첫 문서 완성 (3분 이내)
   └─> 내용 입력/편집 완료
   └─> Ctrl+S 또는 "다운로드" 버튼으로 저장
   └─> [성공 지표 달성] ✓
```

### 페르소나별 핵심 플로우

#### 1. 개발자 - README 작성 시나리오
```
목적: GitHub README.md 작성 및 미리보기

플로우:
1. 사이트 접속 → "README 템플릿" 선택
2. 프로젝트 정보 입력 (제목, 설명, 기능, 설치 방법 등)
3. 코드 블록 추가 (Syntax Highlighting 확인)
4. 이미지 드래그 앤 드롭 삽입
5. GitHub 스타일로 프리뷰 확인
6. .md 파일 다운로드 → GitHub에 업로드

핵심 기능:
- GitHub 스타일 프리셋
- 코드 블록 하이라이팅
- 이미지 삽입
- README 템플릿
```

#### 2. 초보자 - 마크다운 학습 시나리오
```
목적: 마크다운 문법 학습 및 연습

플로우:
1. 사이트 접속 → 환영 문서에서 기본 문법 확인
2. "문법 치트시트" 열기 → 카테고리별 예시 확인
3. 에디터에 직접 입력하며 실시간 결과 확인
4. 툴바 버튼 사용 → 클릭만으로 문법 삽입
5. 자동완성 체험 (### 입력, ``` 입력 등)
6. 완성된 문서 다운로드

핵심 기능:
- 초기 플레이스홀더 (문법 가이드)
- 툴바 버튼
- 문법 치트시트
- 자동완성
- 실시간 프리뷰
```

#### 3. PM/기획자 - 기획서 작성 시나리오
```
목적: 제품 기획서/스펙 문서 작성

플로우:
1. 사이트 접속 → "기술문서 템플릿" 선택
2. 헤딩으로 문서 구조 작성
3. 테이블로 요구사항 정리
4. 체크박스로 To-Do 리스트 작성
5. Mermaid로 플로우차트 삽입
6. TOC 사이드바로 네비게이션 확인
7. PDF로 내보내기 → 팀원과 공유

핵심 기능:
- 기술문서 템플릿
- 테이블 지원
- 체크박스 (GFM)
- Mermaid 다이어그램
- TOC 자동 생성
- PDF 내보내기
```

#### 4. 블로거 - 포스트 작성 시나리오
```
목적: 블로그 포스트 초안 작성

플로우:
1. 사이트 접속 → "블로그 포스트 템플릿" 선택
2. 제목, 부제, 날짜 입력
3. 본문 작성 (헤딩, 리스트, 링크 등)
4. 이미지 클립보드 붙여넣기 (Ctrl+V)
5. Notion 스타일로 프리뷰 확인
6. HTML 내보내기 → 블로그 에디터에 붙여넣기

핵심 기능:
- 블로그 포스트 템플릿
- Notion 스타일 프리셋
- 이미지 클립보드 삽입
- HTML 내보내기
```

## 경쟁 제품 대비 차별점

| 차별점 | 설명 |
|--------|------|
| **설치 불필요** | 브라우저만 있으면 바로 사용 (vs Typora, VS Code) |
| **스타일 다양성** | GitHub/Notion/VS Code/미니멀 스타일 선택 (vs 단일 스타일) |
| **확장 문법** | 수식(KaTeX) + 다이어그램(Mermaid) 기본 지원 |
| **가벼움** | 빠른 로딩, 즉시 사용 가능 |
| **오프라인** | PWA로 인터넷 없이도 사용 가능 |

## 주요 기능

### 1. 파일 입력 방식

| 기능 | 설명 |
|------|------|
| 파일 선택 | 파일 탐색기에서 .md 파일 선택 |
| 드래그 앤 드롭 | 파일을 드롭존에 끌어다 놓기 |
| 직접 입력 | 텍스트 에디터에서 마크다운 직접 작성 |

### 2. 마크다운 렌더링

#### 기본 문법
- 헤딩 (h1-h6)
- 볼드, 이탤릭, 취소선
- 순서 있는/없는 리스트
- 링크, 이미지
- 인라인 코드, 코드 블록 (syntax highlighting)
- 인용문
- 수평선

#### GitHub Flavored Markdown (GFM)
- 테이블
- 체크박스 (task list)
- 자동 링크
- 취소선 (`~~text~~`)

#### 확장 기능
- **수식**: KaTeX 지원
  - 인라인: `$E = mc^2$`
  - 블록: `$$\sum_{i=1}^n i = \frac{n(n+1)}{2}$$`
- **다이어그램**: Mermaid 지원
  - 플로우차트
  - 시퀀스 다이어그램
  - 클래스 다이어그램
  - 간트 차트 등

### 3. UI/UX

#### 레이아웃 (데스크톱)
```
┌─────────────────────────────────────────────────────────────────┐
│  📄 파일명.md    [☀️/🌙] [⚙️] [⛶]          Markdown Preview    │
├───────┬────────────────────────┬────────────────────────────────┤
│       │                        │                                │
│  TOC  │      에디터 영역        │       프리뷰 영역              │
│       │    (마크다운 입력)      │    (렌더링된 결과)             │
│ [접기]│                        │                                │
│       │         ◀ 드래그 ▶     │                                │
└───────┴────────────────────────┴────────────────────────────────┘
```

#### 헤더 구성
| 요소 | 설명 |
|------|------|
| 파일명 | 현재 열린 파일명 표시 (미저장 시 * 표시) |
| 테마 토글 | 다크/라이트 모드 전환 버튼 |
| 설정 버튼 | 폰트, 스타일, 레이아웃 등 설정 패널 열기 |
| 전체화면 | 에디터/프리뷰 영역 전체화면 확대 |

#### 레이아웃 옵션
- **데스크톱**: 좌우 분할 (기본), 드래그로 비율 조절 가능
- **모바일**: 사용자 선택 가능
  - 탭 전환 (에디터 ↔ 프리뷰)
  - 상하 분할
  - 좌우 분할 (가로 모드)
  - 프리뷰만 보기

#### 목차 (TOC) 사이드바
- 왼쪽에 헤딩 기반 목차 자동 생성
- 펼치기/접기 토글 버튼
- 클릭 시 해당 위치로 스크롤
- 현재 보고 있는 섹션 하이라이트

#### 주요 UX 요소
- **실시간 프리뷰**: 입력과 동시에 렌더링 (debounce 300ms 적용)
- **스크롤 동기화**: 에디터와 프리뷰 스크롤 연동 (토글로 ON/OFF)
- **다크/라이트 모드**: 시스템 설정 따르기 기본, 수동 전환 가능
- **반응형**: 화면 크기에 따라 레이아웃 자동 조정

### 6. 데이터 저장/관리

#### 저장 방식
| 방식 | 설명 |
|------|------|
| 로컬 다운로드 | .md 파일로 로컬에 저장 |
| 브라우저 저장 | localStorage에 임시 저장 (자동 저장) |
| 클라우드 연동 | 외부 클라우드 서비스와 동기화 |

#### 클라우드 연동 (v1.1+)
- Google Drive
- Dropbox
- GitHub (Gist, Repository)
- OneDrive

#### 버전 관리
- **자동 저장**: 30초마다 또는 500자 입력마다 자동 저장, 최근 10개 버전 이력 유지
- **수동 스냅샷**: 사용자가 원할 때 버전 저장 (최대 50개)
- **복원**: 이전 버전으로 되돌리기 (diff 하이라이팅 제공)

## 디자인 요구사항

### 1. 테마 시스템

#### 기본 동작
- **시스템 설정 따르기**: OS 다크/라이트 모드 감지하여 자동 적용 (기본값)
- **수동 전환**: 헤더의 토글 버튼으로 강제 전환 가능
- **설정 저장**: localStorage에 사용자 선택 저장

#### 프리뷰 스타일 프리셋
사용자가 선택 가능한 4가지 스타일:

| 스타일 | 설명 |
|--------|------|
| **GitHub** | GitHub README 스타일. 익숙하고 가독성 좋음 |
| **Notion** | Notion 문서 스타일. 깔끔하고 모던함 |
| **VS Code** | 코드 에디터 스타일. 개발자 친화적 |
| **미니멀** | 최소한의 스타일. 여백 많고 심플함 |

#### 커스터마이징
- **전체 테마 커스터마이징 지원**
  - 배경색, 텍스트 색상
  - 헤딩 색상, 링크 색상
  - 코드 블록 테마
  - 강조 색상 (액센트)

### 2. 타이포그래피

#### 에디터 폰트

## 비기능 요구사항

### 성능
#### 파일 크기별 렌더링 성능
| 파일 크기 | 목표 렌더링 시간 | 처리 방식 |
|-----------|------------------|-----------|
| ~1MB | 100ms 이하 | 즉시 렌더링 |
| 1MB~5MB | 2초 이내 | 가상화 적용 (virtual scrolling) |
| 5MB~10MB | 5초 이내 | 경고 메시지 + 청크 로딩 |
| 10MB 이상 | 차단 | "파일이 너무 큽니다" 에러 메시지 |

#### 응답 성능
- **초기 로딩**: 3초 이내 (Lighthouse 기준)
- **프리뷰 debounce**: 300ms
- **스크롤 동기화**: 60fps 유지
- **파일 다운로드**: 1초 이내 (1MB 기준)

#### 성능 최적화
- Code splitting으로 초기 번들 크기 최소화
- 이미지 lazy loading
- Mermaid/KaTeX 동적 로딩 (사용 시에만)
- Service Worker 캐싱
- **목표**: Lighthouse 성능 점수 90점 이상

### 접근성
#### 키보드 네비게이션
- Tab 키로 모든 인터랙티브 요소 접근 가능
- 포커스 인디케이터 명확히 표시
- Skip to content 링크 제공
- 모달/드롭다운에서 Esc 키로 닫기

#### 스크린리더 지원
- 모든 버튼/링크에 의미있는 aria-label
- 랜드마크 역할(role) 명시 (main, navigation, complementary)
- 동적 콘텐츠 변경 시 aria-live로 알림
- 이미지에 alt 텍스트 필수

#### 시각적 접근성
- 색상 대비 비율 4.5:1 이상 (WCAG AA)
- 고대비 모드 지원
- 폰트 크기 12px~24px 조절 가능
- 200% 확대 시에도 레이아웃 유지

#### 수락 기준
- [ ] WAVE 도구 검사 0개 에러
- [ ] 키보드만으로 모든 기능 사용 가능
- [ ] NVDA/VoiceOver 스크린리더 테스트 통과
- [ ] WCAG 2.1 AA 준수 (axe DevTools 검증)

### 브라우저 호환
- Chrome, Firefox, Safari, Edge 최신 2개 버전
- 모바일: iOS Safari, Android Chrome

### 오프라인 지원
- PWA로 오프라인에서도 완전한 기능 사용
- Service Worker로 에셋 캐싱
- 오프라인 시 자동 저장 → 온라인 복귀 시 동기화

### 보안
- XSS 방지 (마크다운 렌더링 시 sanitize)
- 로컬 저장 데이터 암호화 옵션

## 예외 상황 처리 (Edge Cases)

### 파일 크기 제한
| 시나리오 | 동작 |
|----------|------|
| 파일 크기 10MB 초과 | "파일이 너무 큽니다. 10MB 이하 파일만 지원합니다." 에러 메시지 표시 |
| 빈 파일 업로드 | 빈 에디터 표시 (에러 없음) |
| 파일 읽기 실패 | "파일을 읽을 수 없습니다. 다시 시도해주세요." 에러 메시지 |

### 브라우저 저장소 제한
| 시나리오 | 동작 |
|----------|------|
| localStorage 용량 초과 (5MB) | 가장 오래된 자동 저장 항목부터 삭제 |
| localStorage 비활성화 | "브라우저 저장소가 비활성화되어 자동 저장이 불가능합니다." 경고 + 다운로드 권장 |
| IndexedDB 미지원 | localStorage로 폴백 (제한된 버전 관리) |

### 렌더링 오류
| 시나리오 | 동작 |
|----------|------|
| 잘못된 Mermaid 문법 | 에러 메시지 표시 + 원본 코드 블록으로 폴백 |
| 잘못된 KaTeX 수식 | "수식 오류" 빨간색 텍스트 표시 + 원본 표시 |
| 깨진 이미지 링크 | 대체 텍스트(alt) 표시 + 이미지 아이콘 |
| 렌더링 타임아웃 (10초) | "렌더링 시간 초과" 경고 + 부분 렌더링 결과 표시 |

### 네트워크 오류 (v2.0)
| 시나리오 | 동작 |
|----------|------|
| 클라우드 업로드 실패 | 3회 재시도 → 실패 시 로컬에 임시 저장 + 알림 |
| 클라우드 다운로드 실패 | "파일을 불러올 수 없습니다" 에러 + 캐시된 버전 제공 (있을 경우) |
| 오프라인 상태 | "오프라인 모드" 배너 표시 + 로컬 기능만 사용 가능 |

### 동시 편집 충돌
| 시나리오 | 동작 |
|----------|------|
| 여러 탭에서 동일 문서 편집 | localStorage 이벤트로 동기화 (마지막 저장 우선) |
| 다른 기기에서 동시 편집 (v2.0) | 충돌 감지 → "다른 곳에서 수정되었습니다" 경고 + Diff 모달 표시 |
| 버전 충돌 | 사용자에게 "현재 버전 유지" vs "서버 버전으로 덮어쓰기" 선택 제공 |

### 사용자 입력 오류
| 시나리오 | 동작 |
|----------|------|
| 지원하지 않는 파일 형식 (.pdf, .docx 등) | ".md 파일만 지원합니다" 에러 메시지 |
| 파일 드래그 중 취소 | 조용히 무시 (에러 없음) |
| 붙여넣기 실패 (권한 없음) | "클립보드 접근 권한이 필요합니다" 안내 |

### 브라우저 호환성 이슈
| 시나리오 | 동작 |
|----------|------|
| 구형 브라우저 (IE11 등) | "이 브라우저는 지원하지 않습니다. Chrome, Firefox, Safari, Edge를 사용해주세요." 페이지 표시 |
| Service Worker 미지원 | PWA 기능 비활성화 + "오프라인 지원 불가" 알림 |
| File API 미지원 | "브라우저가 파일 업로드를 지원하지 않습니다" 에러 |
